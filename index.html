AI Engineering

I Build Agentic AI Systems

We're in the age of AI and I build the agents that power it. From autonomous, tool-using agents and multi-agent orchestration to RAG pipelines and production LLM systems, I design AI that reasons, uses tools, and takes real action.

Autonomous AI Agents

I design goal-driven agents that plan, reason, and execute multi-step tasks using tool-calling, memory, and self-correction loops (agents that get work done, not just chat).

Multi-Agent Orchestration

I build collaborative agent systems where specialized agents coordinate, delegate, and verify each other's work using orchestrator–worker patterns and graph-based workflows (LangGraph).

RAG & Knowledge Systems

I ground LLMs in private data with retrieval-augmented generation, vector search, embeddings, and semantic memory so answers are accurate, sourced, and trustworthy.

Tool Use & MCP

I connect agents to real-world tools, APIs, and the Model Context Protocol (MCP) so they can take action across systems, booking, querying, automating, and integrating end-to-end.

LLM Engineering & Prompting

Production prompt engineering, structured outputs, function calling, evaluation, and guardrails across OpenAI, Anthropic Claude, and open-source models (reliable AI you can ship).

Agentic 3D & Spatial AI

My frontier: fusing LLM agents with XR and real-time 3D AI that generates, composes, and drives immersive spatial experiences. It's the direction powering our WebXR platform, Relu Spatial.

My Process

How I Build Production AI Agents

01
Define Goal & Guardrails

Clarify the task, success criteria, scope, and safety boundaries the agent must operate within.

02
Design Tools & Context

Wire up the right tools, APIs, RAG retrieval and MCP servers so the agent has the context to act.

03
Orchestrate Reasoning

Compose planning, memory, and multi-agent workflows, single agent or coordinated swarm.

04
Evaluate & Trace

Build evals, trace every step, catch failures, and iterate on prompts, tools and routing.

05
Ship & Monitor

Deploy to production with logging, cost controls, and observability then keep improving.
